# 부최박 팀

## 기본 요구 사항
- 사용자 구분 필요
- 폴더 파일 존재
- 폴더 Create, Read, Delete
- 파일 Create, Read, Delete
- 파일 이동, 폴더 이동, 파일 복사, 폴더 복사

## 추가 요구사항
- 동시에 여러명이 업로드/다운로드를 요청할 때 적절한 정책/방법 존재해야함
  - 4core 8gb 서버에서 최대한 많은 사용자가 동시에 업로드/다운로드 할 수 있게 구현
    - 서버가 감당할 수 없는 요청이 들어올 때 적절한 에러 처리 혹은 대기열 관리 필요
- 업로드 시간도 관리할 것인가?
- 다운로드 기록 관리할 것인가?
- 업로드 날짜에 따라서 데이터 검색이 가능한가?
- 스토리지 전체 용량이 가득 찼을 때 정책 설정 필요
  - (압축, 오랜된 데이터 삭제 등)
    - 업로드를 제한할 것인가? 허용할 것인가?

## 추가 의견
- 실제 데이터를 분할해서 관리할 것 인가?
- 다운로드 기록을 바탕으로 cache layer 구성을 가져갈 것인가?
- 다운로드 기록을 바탕으로 데이터 압축률 조정할 것 인가?

## Edge Case
- 바이너리가 완전히 동일한데 이름/ 경로만 다른 데이터 다수 일 때
- 일부데이터만 수정된 데이터가 동일한 경로에 올라갈 때

## how
- id, 물리적 path, 논리적 path 구조 사용
- 논리적 path에 대해서는 b+tree 구조 혹은 유사한 데이터 구조 사용
  - 고려해야할 내용
    - nosql 디비를 활용

- 메타 데이터를 관리할 2차원 테이블
  - col: 날짜, 논리 경로, 메타2, 메타3, 파일 헤시, 물리 경로, 사용자 id
